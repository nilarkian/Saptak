00 — System Overview

Three Tiers, One Router, ~95% Savings

Most AI cost waste comes from one mistake: using expensive models for cheap tasks. This system routes every call to the cheapest model that can handle it — free local for mechanical work, cheap API for reasoning, premium only for identity-critical decisions.

🎯
Route First

Three tiers: local/free → cheap API → premium. Classify the task, dispatch to the cheapest competent tier, escalate on failure. Never fan out to multiple tiers — route once.

⚖️
Two Invariants

Quality floor: no compression ships if output gets worse. Permanent memory: long-term memory survives every optimization pass. Both override every other law.

💰
~93–95% Savings

Router alone: 67–90% cost reduction (30% free local + 40% cheap + 30% premium). Output doctrine adds 3–10% more. Combined: ~93–95% vs all-premium baseline.

01 — Tier Architecture

The Three Tiers

Each tier has a class, a model, a cost profile, a call-distribution target, and token budgets for prompts and outputs. The quota window is 50 rolling calls; tolerance is ±10%. Hard-floor purposes never get demoted by quota logic.

Tier Model & Endpoint Target / Budgets
T1 — Chat qwen3:32b via local Ollama (free) 30% target · ≤200 prompt tokens · ≤100 output tokens. RAM-pick: >24 GB → 32b · 12–24 GB → 14b · <12 GB → 3b
T2 — Cheap deepseek-v4-pro → deepseek-v4-flash (cascade) via api.deepseek.com 40% target · ≤800 prompt tokens · ≤500 output tokens. Auto-scaled timeout: 90s + 1s/1K input chars, cap 300s. 1 retry on 429/502/503/504/AbortError.
T3 — Precision ★ claude-opus-4-7 via api.anthropic.com 30% target · ≤1500 prompt tokens · task-dependent output. Hard-floor purposes always land here regardless of quota. T3 fail → throw (no further fallback).
⚑ Cascade Order T1 fail → T2. T2-pro fail → T2-flash → T3. T3 fail → throw. Never fan out in parallel — route once, escalate on failure only. Parallel calls multiply cost and rarely improve quality.
02 — Routing Keywords

Purpose Buckets

The classifier checks purpose first, then flags, then prompt heuristics. Hard-floor purposes skip all quota logic and route directly to T3 regardless of rolling distribution. Everything with no matching purpose or flag defaults to T3.

03 — Setup Protocol

Five Phases to Production

Run the v2 compressed prompt in Claude Code (or any agent with bash + file write). ~5–10 min on broadband. The only interaction required: paste API keys when asked.

A
Environment Check node --version ≥18. Install Ollama if missing (winget/curl/brew). Probe http://localhost:11434/api/tags — run ollama serve if not responding. Pick T1 model by free RAM: >24 GB → qwen3:32b, 12–24 GB → qwen3:14b, <12 GB → llama3.2:3b. Pull chosen model.
B
Keys Read existing .env. For each missing key, ask once — never echo after capture. Write DEEPSEEK_API_KEY (platform.deepseek.com → API Keys, sk-…) and ANTHROPIC_API_KEY (console.anthropic.com, sk-ant-…) plus all 11 config vars to .env. Append .env to .gitignore.
C
Files — Write 7 CommonJS modules lib/soft-failure.cjs — append-only soft failure log. lib/ollama-client.cjs — T1 caller + health probe. lib/deepseek-client.cjs — T2 caller with auto-scaled timeout + retry. lib/anthropic-client.cjs — T3 caller. lib/deepseek-verify.cjs — hallucination guard (file/path/SM-number checks). lib/tiered-ask.cjs — main router (classify → quota → dispatch → log). scripts/tier-usage-report.cjs — distribution + latency + fallback stats.
D
Verify — 6 checks in order node lib/tiered-ask.cjs ping → all 3 tiers ok:true. T1 routing test (prompt: "hi" → tier 1). T2 routing test (purpose: summarize → tier 2). T3 routing test (long design prompt → tier 3). Hard-floor test (purpose: identity_audit → tier 3). node scripts/tier-usage-report.cjs → distribution table. Report failure with exact next step; never fake success.
E
Report Print T1/T2/T3 model + latency. Quota config (window size, targets, tolerance). Paths to lib/tiered-ask.cjs and scripts/tier-usage-report.cjs. Usage: const { ask } = require('./lib/tiered-ask.cjs') then ask({ prompt, purpose }).
04 — Non-Negotiables

Rules That Don't Bend

Six rules that override operational convenience. Violating any one breaks the system's trust guarantees or data integrity.

The Six Invariants

These apply in all modes — setup, production, maintenance, compression passes. No exception without explicit user confirmation and logged rationale.

  • Never Echo Keys API keys are written to .env once. Never logged, never re-displayed, never emitted in outputs after initial capture.
  • Never Delete Logs memory/tier-usage.jsonl and memory/soft-failures.jsonl are append-only. Rotation requires a backup first. Silent deletion is data loss.
  • Never Modify Hard-Floor The HARD_FLOOR purpose set in tiered-ask.cjs is not changed without explicit user confirmation. It protects identity-critical calls from cost pressure.
  • No Output Padding No preambles, postambles, plan narration, or acknowledgment tokens in any output — at any tier. Strip on sight, every time.
  • Hard-Floor Always T3 Quota rebalancing, cost pressure, and cascade logic cannot demote a hard-floor purpose from Tier 3. The classifier check happens before quota math.
  • No Silent Eviction Long-term memory entries are only removed via deliberate, logged action. "Tightening" a prompt that drops memory entries is not compression — it is amnesia.
📐 The Doctrine — Operating Laws
05 — Output Doctrine

Five Laws of Output Discipline

Output is the most expensive surface — every emitted token is billed. These five habits close the four leaks that burn ~30% of output tokens in unoptimized prompts: narrated reasoning, restated instructions, forward-carried context, and confirmatory tool calls.

1

No Preambles, No Postambles

Trigger: every output, every tier, every call.

The Habit Start with the answer. Emit the artifact. Stop when done.
The Rule Strip on sight: "Sure!", "Great question", "I'll first…", "Let me know if anything's unclear", "Hope this helps." Zero value, measurable cost.
2

Structure Beats Prose

Trigger: any list, comparison, sequence, status, or number.

The Habit Use tables for comparisons, JSON for structured state, IDs for references, symbols for statuses (✓ / ✗ / ⚠). "~20k" not "approximately twenty thousand."
The Rule Prose is for nuance that structure can't carry. If structure can carry it, it must. Narrative sequences with state → state machine, not sentences.
3

Route Before Reasoning

Trigger: before any LLM call. Before any tool call.

The Habit Ask: is the answer in cache? Computable directly? A fixed transformation (regex/parse/format)? If yes to any — don't call the model. If no — classify tier first, dispatch to cheapest competent.
The Rule Escalation, not parallelism. Route once; escalate on failure. If the quota is constantly demoting calls, the classifier is wrong — fix the classifier, not the quota.
4

Reference Over Inline

Trigger: any content >200 tokens, or content that changes, or content only some steps need.

The Habit Pass path/ID/hash/stub. Load full payload only when a step has explicitly decided it needs it. Lazy-load tiers: identifier only (~10 tokens) → summary stub (~50 tokens) → full payload.
The Rule Carrying context the next step won't read is pure waste, charged on every turn until eviction. "Might be useful" = "probably isn't" — the optionality costs real tokens.
5

Deltas, Not Snapshots

Trigger: file edits, state changes, memory updates, task completions.

The Habit Output what changed, not the full state. "Edited 3 lines: …" not the full file. "10/10 done; 1 failed" not a transcript. "Memory: 1 updated" not the new memory file.
The Rule Deltas every turn; snapshots at session boundaries or every N deltas (N large enough to amortize snapshot cost). Idempotent deltas — applying twice = same result as once.
6

Tool-Call Economy

Trigger: before any tool call, and before re-injecting tool output into context.

The Habit Pass minimum args. Batch N calls that share setup cost into one. Distill tool output before re-injecting — extract the 3 fields, drop the 5K blob. Cache hot results (stable-for-minutes = wrap in cache).
The Rule Skip the call if: answer is in this turn's context, in fresh long-term memory, computable directly, or tied to no decision. "Exploratory" calls with no decision criterion are theater — and billed.
06 — Memory Architecture

Three Memory Tiers

Memory is leverage when curated and cost when hoarded. Each tier has a distinct lifespan, eviction policy, and promotion rule. The worst mistake: letting working-memory content "graduate" automatically, turning long-term into a junk drawer.

Long-Term — Permanent Survives sessions, model swaps, prompt rewrites. Holds: identity facts, earned learnings, decision rationale (with why), self-modification log, unresolved tensions, index pointers. Sacred. Eviction is deliberate + logged — never a side effect. Read on start; write earned signal on close. Always.
Working — Session-Scoped Survives the session. Evicted at session end. Holds current task's load-bearing facts. Close ritual: summarize in 3–5 lines → promote durable signal to long-term → discard the rest. A session that closes without writing is a session that lost.
Turn-Local — Ephemeral Survives the turn. Auto-evicted. Tool results, intermediate computations, drafts superseded this turn. Never promoted automatically — promotion is always intentional. The "just in case" trap fires here: carrying it forward charges every subsequent turn.

"Long-term memory survives every compression. Compression that breaks this is not compression — it is amnesia."

07 — Operating Heuristics

The 22 Laws — Five Clusters

The full doctrine compresses to 22 operating heuristics, 15 words or fewer each. The two invariants (quality floor, permanent memory) override all 22. These five clusters cover the laws that cause the most silent failures when violated.

1

The 95% Test

Every Turn
  • For every line of prompt or token of output: "Does removing this degrade behavior?"
  • If no — delete it. If yes — what is the smallest version that preserves behavior? Use that.
  • Apply recursively: paragraph → sentence → clause. Apply to whole sections too.
  • False-positive trap: "feels important" ≠ behavioral change. Pattern-matches to professional writing, costs like real content. Cut it.
Outcome

Operating prompts shrink 3–5×. A 2,000-token prompt usually wants to be 400–700 tokens.

2

Context Decay

Every Session
  • Anything past N turns ago is effectively forgotten unless explicitly re-surfaced.
  • Don't carry stale context forward — re-fetch when the next step actually needs it.
  • First instruction in a long prompt has more weight than the middle. Put the most important rules first.
  • Rewrite negative rules as positive ones. "Don't do X" is weaker than "always do Y."
Outcome

Prompts that "evolved" over time usually grew. Token count over time is the fastest way to find prompt rot.

3

Variance Is Expensive

As Needed
  • A prompt that works 95% of the time and breaks 5% costs more than 90%-quality output 100% of the time.
  • Failures are silent unless instrumented. Log every tier failure — the tier-usage.jsonl is that instrument.
  • If a tier consistently over-represents in the quota window, first check: are the prompts going to that tier bloated?
  • Bloated prompts route up. Trimmed prompts route accurately. Fix the prompts before tuning the classifier.
Outcome

90%/100% beats 95%/95%-broken. Reliability is cheaper than ceiling quality.

4

Memory Contract

Every Session
  • Read long-term memory on start. Always. A session that starts cold is a session that drifts.
  • Write earned signal on close. Always. Summarize-then-discard for working state; promote durable signal to long-term.
  • Dedup on write — a new entry that re-says an existing one updates the existing one, never appends a duplicate.
  • Permanence test: can a future agent (next week, after a model swap) open long-term memory and reconstruct who it is, what it knows, what it decided? If anything is missing, the contract is broken.
Outcome

Identity, earned learnings, and decision rationale persist across every prompt rewrite and model swap.

5

Honest Accounting

Before Shipping
  • Never report a compression ratio without computing it. Char-count ÷ char-count. Ten seconds to run; catches every fake number.
  • Three deltas, reported separately — never netted: compression (−X) = same content, fewer tokens. Permanent-memory (+Y) = new long-term entries, intentional. Doctrine-install (+Z) = new rules, paid forward. A positive net is fine when Y or Z accounts for it; never call it "compression."
  • Every doctrine install pairs with a measurable downstream metric, a review window (14–30 days), and a rollback condition tied to the metric — not to feel. If the metric doesn't move after the window, roll back.
  • If the only number you report is the net, you are either lying or sloppy. Both are bugs.
Outcome

"Measure twice, report three numbers, never net them." A compression that wins tokens but loses correctness is not a win.

08 — Conclusion

Strip Until It Breaks, Then Stop

The router solves the tier problem: which model handles this call? The doctrine solves the token problem: what does every tier send, store, and emit? Neither works without the other. The router without doctrine ships bloated prompts to cheap models and undoes half its own savings. The doctrine without the router applies compression pressure on premium calls that should have been routed away entirely.

The 94% headline requires both halves. The two invariants — quality floor and permanent memory — are not constraints on the system; they are the reason the system is trustworthy enough to run autonomously. Every compression that violates them is not compression. It is either a cheaper answer or a forgotten self.

Route cheap. Compress outputs. Keep memory clean. Strip until removing more would degrade. Stop one step earlier.

⚑ The One Rule Before any LLM call: could this be cached, computed, or templated? If yes — don't call the model. If no — classify tier first, dispatch to cheapest competent, escalate on failure only. Output the answer and nothing else.
